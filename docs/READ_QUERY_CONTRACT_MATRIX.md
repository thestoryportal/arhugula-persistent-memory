## Headline reading of the read slice
- **What the model/weights actually owe (WEIGHTS):** R5 native-knowing + R15 constraint-firing + the write-half of R9 delete — i.e. *does committed knowledge fire reliably, and can it be removed*. The reliability-at-scale question (R5) is the live falsifier → **folded into compaction-CORE-at-scale** (council #1), not a standalone B1.
- **What the spec DELEGATES off the weights (GOV/HYBRID):** R1 SELECT, R6 closed-world/commit-status, R10 consistency, R11 medium-authority, R14 oracle, R16 provenance. B0 confirmed R6 is governance-delegated — *not a weight falsification.*
- **Sharp UNTESTED falsifiers worth GPU later:** R2 reverse-lookup (reversal-curse), R9 deletion-non-resurfacing, R15 constraint-probe.
- **Spec GAPS flagged for F1 (under-specified read surface):** R6 read-time provenance/closed-world; R7 aggregation/negation; R8 transitive semantics; R13 L1-vs-L2 split. The spec specifies read *requirements* (exact/reverse/traverse) but ships no formal query-language section — the least-specified production surface (confirms the e2e §G note).
