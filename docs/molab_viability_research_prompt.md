# Perplexity deep-research prompt — does molab (marimo) work for our MEMIT-editing program?
_Authored 2026-06-17. Paste into Perplexity. Goal: surface every decision-relevant fact before piloting a move off the paid RunPod RTX 4090 to molab's free RTX Pro 6000._

> **Role & goal.** You are a research assistant evaluating whether **molab** (marimo's hosted notebook service, molab.marimo.io) can host a specific GPU research workload. Answer with **primary sources** — marimo's official docs/blog, the molab FAQ/terms, NVIDIA spec sheets, PyTorch release notes, Cloudflare R2 docs — and cite each claim with its source and strength (high = official/primary, medium = indirect, low = inference). Explicitly flag anything unverified or undocumented, and SEO/marketing blogs should not be used as evidence.
>
> **Our workload (the thing that must run).** In-weight, MEMIT-class knowledge editing of LLMs in PyTorch + HuggingFace transformers (fp16): load 3B–8B models (GPT-J-6B, Qwen2.5-7B, Llama-3.2-3B, possibly Gemma 4 12B–31B), precompute and CACHE per-layer covariance statistics (~1 GB .npz each, several per model), then run many sequential weight-edit experiments. Characteristics: (a) a **persistent working tier of tens of GB** (model weights 10–30 GB each + covariance caches + artifacts; total easily 50–150 GB); (b) **long unattended jobs**, 30 min to 3+ hours, that we currently run as detached `setsid`/`nohup` background processes surviving terminal disconnects; (c) an autonomous **Claude Code CLI agent** driving the work from a terminal; (d) strict reproducibility gates (bit-exact cross-session determinism, known-baseline reproduction). Current host: paid RunPod RTX 4090 (24 GB). Candidate: molab free tier (RTX Pro 6000 Blackwell 96 GB, 4 CPU, 32 GB RAM, 12 h sessions, 90 min idle shutdown, "limited persistent storage" on Cloudflare R2, marimo notebooks not Jupyter, with a terminal that can run Claude Code).
