## 1. THE DETERMINATION (stated once)

> **The LLM-as-Database spec is NOT-READY-WITH-CONDITIONS.** Its **batch/Genesis write-and-deploy spine is implementable and proven-for-scope** at **3B / N≤100 / band[4-8] / single-batch / mostly 1-seed / fictional-domain / AlphaEdit**. Outside that envelope the determination is **OPEN, with several first-class falsifiers (§3) that could still defeat implementability.** The **read contract is now CHARACTERIZED (not delivered) along the weights↔index seam** (2026-06-25 slice): **(a) weight-native native-knowing** is robust **for no-pretrained-prior facts** (insert + re-edit; the realistic code/structural workload) and fragile-but-recipe-rescuable against entrenched pretrained priors (R5/R13/R5b); **(b) the structured-query half** (reverse, deletion, closed-world) is **confirmed NOT weight-native (R2/R6/R9/§11.2) → its obligation RELOCATES to the `.vindex`/index/query layer, which is prototyped-not-empirical and largely unbuilt** (R1/R4/R7/R8/R11/R16 untested). The sharpest open falsifiers remain **compaction-at-true-scale** (stimulus-gated), the **index/query layer** (now load-bearing), and the **governance/security/pruning** layers (design-viable, not empirically proven, not red-teamed).

## 2. WHAT IS SETTLED (pointers — not restated)

**A. The structural anchor (verified against the primary spec this session).**
**§11.2/D42 — NO content class is weights-authoritative.** The spec names exactly two authoritative media: **Git** (for `structural_entity`) and **`.vindex`** (the overlay *file*, for Genesis-L4 `domain_concept` + `constraint_rule`). Both are on-disk, versioned artifacts. The **live weights are always the *serving copy*** — compiled from `.vindex` at mount and re-derivable from `.larql` patches via §8.10 compaction (= full MEMIT re-run). ⚠ *Nuance:* `.vindex`-authoritative ≠ weights-authoritative (`.vindex` is a file; weights are the served copy). **Consequence:** in-weight storage is *never* the source of truth → the only genuinely **weights-owned** obligations are **behavioral** (R5 native-firing, R15 constraint-firing), NOT authoritative-storage. This strengthens the scope-keyed hybrid (D-B3N-1) and is the spine of the read-contract findings below. (Verified: spec lines 633–638; `[[in-weight-falsifier-must-be-weights-owned]]`.)

**B. Proven-for-scope (the data-path spine).** All at scope tuple above; A1 re-graded *constructive-at-one-point* (B3N), not a universal law.
- Batch/Genesis compile is corruption-clean — **A1** (`D-A1-1`, CORPUS/14).
- Survives real Q4_K_M quantization (a *mix*, stricter than LARQL's uniform-Q6_K serve) — **B3** (`D-B3-1`, CORPUS/17; `[[q4km-quantization-survival-pass]]`).
- Serves edited facts on commodity CPU via llama.cpp — **E1 Claim A** (`D-E1-1`, CORPUS/18). *(LARQL `gguf-to-vindex` ingest FALSIFIED for Qwen2.5 — bias-drop, A7-causal; model-family split.)*
- Recipe = D20-aligned in-solve AlphaEdit (null-space P + preserve-sampling + batched compile); is the spec-MANDATED safeguarded-MEMIT family — **CP3/D12** (CORPUS/09).

**C. Amended (spec text must change).**
- **§8.7 drift contract** — the spec's count-only `edge_count_since_anchor` is the **wrong predictor**; corruption is **relation-concentration- + edit-order-dominated** → concentration-aware `max_relation_concentration_since_anchor`, worse-of vs global count, conservative `k≤1`. Model-general (3B+7B). **D1/B1/D-D1-2** (CORPUS/22; `docs/SPEC_8_7_AMENDMENT_DRIFT_CONCENTRATION.md`, operator-approved). *Numeric value 3B-set; 7B cross-model transfer = OQ-W1, OPEN (Condition 4).*

**D. Decided (reasoned architectural position, not an empirical PASS).**
- **In-weight is NOT contractually required** (§11.2 makes it a serving copy; "native knowing" is a stated *preference*, not an enforced read contract). Verdict = **compaction-bounded scope-keyed HYBRID**: in-weight viable for the batch/Genesis core under **3 conditions**; route incremental-high-churn to a gated side-store. **D-B3N-1** (`docs/B3_IN_WEIGHT_NECESSITY_DECISION.md`). The 3 conditions are Conditions 1–3 below.

**E. ⭐ Read-contract + native-knowing slice — CHARACTERIZED along the weights↔index seam (2026-06-25; CORPUS/26–30 + R3 screening).** The 6-experiment slice resolves the read contract into two halves with opposite dispositions:
- **Weight-native behavioral firing (the paradigm payoff) — toward-satisfied, CONDITIONED.** In-weight editing produces *usable, paraphrase-robust* knowledge **for facts with no entrenched pretrained competitor**: NOVEL insert 100% (R5), re-edit of a prior edit 94% (R5b) — the realistic insert/update workload. It is **fragile against dense pretrained priors** (counterfactual-over-prior ~22–25% under paraphrase; failures revert to the true fact — R13/R5), **partially recipe-rescuable** by paraphrase-DIVERSE training (→65%, not intensity). **The discriminating axis is pretrained-prior presence, not edit-count** (R5b). ⚠ **LOAD-BEARING CONDITION (per content-class), not a scope caveat:** code/`structural_entity` facts (fictional-ish identifiers, no common-knowledge overlap) → **robust**; `domain_concept`s that overlap common knowledge → **fragile**. R5's "fictional-domain" robustness *is* this condition.
- **Structured-query reads — confirmed NOT weight-native → obligation RELOCATED to the index, which is unbuilt/prototyped.** Reverse-lookup (R2: write-only, 0/24 reverse), deletion-non-resurfacing (R9), closed-world (R6) are all **index/`.vindex`-delegated** (consistent with §11.2). Relation-family expression screens clean (R3). But L1 deployed `SELECT` read-back (R1, CP2-flagged at-risk), aggregation/negation (R7), multi-hop (R8), namespace (R4), medium-on-read (R11), provenance (R16) are **UNTESTED**, and the index/query layer that must serve them is **prototyped-not-empirical (CP2/G3, self-authored), largely unbuilt** — the SAME epistemic status as governance. **The structured-query burden did not shrink; it moved to an unbuilt layer.**
