3. **Only T.3 changes the finding's class.** The current strongest claim is "config-independent MEMIT-on-base-Llama ceiling." The single experiment that can promote this to (or falsify it as) a broader "base-decoder-LM" claim is a non-Llama architecture. That is the highest-information forward move per unit compute.

## §7'' — Within-regime deeper-arm deprioritization (NEW at v1.2)

### §7''.1 The deprioritized arms

S2.26's matrix pre-named two deeper arms as contingencies: **per-layer sweep** (Berkeley recipe — empirically find Llama-3.2-3B's optimal MEMIT edit layer rather than using either contested band) and **sequential-vs-joint dispatch** (whether 5-fact joint-overlay vs sequential single-fact edits changes the surface). v1.2 formally DEPRIORITIZES both.

### §7''.2 Rationale

| Consideration | Assessment |
|---|---|
| Hypothesis class | Both are config-variation *within* the MEMIT-on-base-Llama regime already 9-axis-confirmed. Each would be a 10th/11th axis of the **same class**, not a new hypothesis class. |
| Prior probability of outcome | High prior of another `0/5` given 9 consistent axes including two config-variation axes (band) and one model-variation axis (scale). |
| Marginal information | Low. A per-layer-sweep `0/5` confirms what Axis 9 already strongly implies (band placement is not load-bearing); a sequential-dispatch `0/5` confirms the joint-batch is not the cause (already implied by the internal-vs-external signature, which is per-fact and not a batch-interference pattern). |
| Information if it CLEARS | This is the one scenario that re-activates them: if T.3 Mistral CLEARS, the per-layer sweep on Llama becomes immediately high-value as a "what's different between the architectures" probe. So the arms are deprioritized, NOT eliminated — they are conditionally re-activated by a surprising T.3 result. |

### §7''.3 Disposition

Per-layer sweep + sequential-vs-joint dispatch: **OPEN, DEPRIORITIZED.** Re-activated conditionally if T.3 produces a CLEAR (ceiling is Llama-specific) rather than a `0/5`. Not scheduled within the current forward plan; T.3 takes priority.
