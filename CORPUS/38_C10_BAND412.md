**Verdict: NO MATERIAL KNOB RESCUE.** The widened `[4,12]` MEMIT/AlphaEdit recipe at the tested strength/hparams does **not** rescue the hard A7 project-coined value class. The binding held-out behavioral read metric worsens (`para_full 11.1 to 6.9`, -4.2pp). Canonical trained-prompt fit improves only modestly (`canon_full 29.2 to 41.7`, +12.5pp), below the pre-registered W-realization-only threshold and far below any usability bar. C10 remains open; this result eliminates this cheap band-widening knob only, not wider-layer recipes in general and not MEMIT capacity.

## The question
CORPUS/36 localized the realistic coined-value failure away from `compute_z`: the z target is achievable at 0.99, so the remaining bottleneck was W-realization. CORPUS/37 then showed a pure later-band `[8,12]` shift did not rescue A7. The operator chose one more cheap W-realization knob before an AnyEdit port. This test asks: **does widening the existing band to `[4,12]` materially improve held-out read expression for hard coined-coined values while preserving A1/A2 controls?**

## Design
Same 24 fictional subjects and A7 coined-coined values as CORPUS/37; relation = capital; binding metric = held-out-paraphrase **full-sequence** exact match across three paraphrases. Two hparam recipes:

- baseline: `configs/hparams/qwen25_3b_memit_hparams.json`, layers `[4,5,6,7,8]`
- wide band: `configs/hparams/qwen25_3b_memit_hparams_band412.json`, layers `[4,5,6,7,8,9,10,11,12]`

Controls were re-run in the same harness: A1 single-token and A2 coherent two-token values. LAW#5 inertness passed for both recipes: baseline `|delta_expr|=0.0001`, `|delta_loc|=0.02`; wide `[4,12]` `|delta_expr|=0.0000`, `|delta_loc|=0.13`.

## What the verdict licenses
**EVIDENCE-SHOWS:** this `[4,12]` recipe at the tested strength/hparams does not materially rescue the hard long no-prior coined-coined value class. The binding §8.9 behavioral-read metric is `para_full`, and it gets worse (`11.1 to 6.9`).

**EVIDENCE-SHOWS:** the run is valid under the pre-registration. Both LAW#5 gates pass, baseline A7 reproduces the failure envelope (`canon_full 29.2`, `para_full 11.1`), and wide-band A1/A2 controls remain above floor (`para_full 90.3` each).

**EVIDENCE-SHOWS:** the widened recipe is not a clean improvement even on controls: A1/A2 remain viable but drop from `100.0` to `90.3`. A7 first-token transport also worsens (`para_first 72.2 to 37.5`) and teacher-forced continuation falls (`56.2 to 40.1`).

**I-INFER:** widening the MEMIT band can modestly improve canonical trained-prompt fit without improving usable paraphrase readout. That is consistent with the C10 failure being a long-value behavioral-read problem rather than merely an under-fit canonical prompt.

**I-INFER:** AnyEdit remains mechanistically justified as a next candidate, but this result does **not** prove AnyEdit necessary. Edit-strength/layer-count remain untested MEMIT-family knobs if the operator wants additional cheap probes.

## Severity / F1 impact
C10 remains **OPEN / CONDITIONAL** and F1 readiness is **UNCHANGED**.
