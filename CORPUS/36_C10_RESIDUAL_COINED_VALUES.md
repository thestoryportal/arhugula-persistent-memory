## What the verdict licenses (advisor + cross-family calibrated — two precise claims)
**(1) C10 the CONDITION (multi-token semantic values fragile in-weight): CONFIRMED and WORSENED.** Realistic project-coined values express at 19–31% held-out-paraphrase full-seq — **below the 85% usability gate, below this run's 2-token incoherent floor (45.8%), and below the original C10 floor (36%)** — because real coined values tokenize LONG (4–5.5 tok) and full-sequence exact-match compounds per-token failure. Lead with this less-flattering absolute frame: **realistic in-weight values are at-or-below C10's contrived worst case in usable expression.**

**(2) C10 the MECHANISM for the realistic class: NOT C10's fit-then-fail-to-generalize parrot — a DISTINCT length-realization wall.** C10's signature was *fit the trained prompt (canon 95.8%) then fail paraphrase (36%)*; the floor arms A3/A4 still show exactly that (canon 95.8%). The realistic arms **fail to even FIT the canonical prompt (canon_full 37–46%)**. The cheap z-probe localizes it: `compute_z` reaches **0.99 target probability for every value class** (single, incoherent, all realistic coined, 3–6 tok), **converged by step 25 (25≈100 steps)** → the failure is NOT z-optimization, NOT z-capacity, NOT a *compute_z* (more-steps) knob. The ideal value vector is perfectly findable; the bottleneck RELOCATES to the **linear W-edit realizing the full multi-token continuation**. ⚠ This rules out the compute_z knob ONLY — the W-realization knobs (band width / layer count / edit strength) are UNTESTED, so option A (AnyEdit) is mechanistically CONSISTENT with the failure, NOT proven NECESSARY; the cheap resolver before a multi-day port = re-run one realistic arm with a wider/later band or more layers and see if canon_full climbs. (Cross-family precise wording: continuation positions are affected only INDIRECTLY via the edited first value-token; there is no DIRECT optimization objective on them → the method is underconstrained for long multi-token strings.)

**Sub-finding (the pre-registered question's clean answer): continuation prior-coherence is mildly PROTECTIVE per-token, overwhelmed by length.** Cite the length-isolated metric — teacher-forced per-continuation-token: realistic arms 60–72% vs incoherent floor 46% (NOT the length-confounded para_full A4→A6/A7 deltas; A4=3 tok, A6=4, A7=5.5 — [[name-the-manipulated-variable-not-the-arm-intent]]). So BPE-predictable sub-words / real category nouns help per position, but cannot rescue full-sequence usability against length compounding.
