When asked to keep documentation in sync / prevent drift (operator pushed hard 2026-06-21: "ALL docs, entire repo, never stale, always auto-updated, otherwise perpetual drift"), the honest engineering truth (Opus advisor + gpt-5.5 cross-family, converged):

- **The literal maximal ask is unsatisfiable.** (1) Nothing can auto-*write* corrected narrative prose — only auto-*detect* + *block*. So the deliverable is **auto-ENFORCED** (no stale commit lands), NOT **auto-updated** (the repo edits itself). Un-conflate these out loud. (2) "Fingerprint every doc" gives the **same false-green** as the presence-only gate it replaced: a fingerprint guards only REGISTERED edges; any unregistered restatement drifts silently. Stamping hash tokens through prose also degrades the readability of the very context docs.
- **The actual cure is DE-DUPLICATION, not instrumentation.** Drift is manufactured by *copying* an evolving result into many docs (this caused the k≤2→k≤1 lapse). Fix the norm: full detail in ONE source (`CORPUS/NN` / amendment / `docs/program_state.json`); every other doc carries a thin self-contained summary = **D-ID + one-line verdict + pointer**, never a restated copy.
- **Right-sized design (built; see `tools/ANTI_DRIFT.md`):** (A) GENERATE what's mechanical (structured status from `program_state.json` via `render_state.py` into `<!-- GENERATED:* -->` blocks) = the real auto-update; (B) FINGERPRINT what's narrative (`closeout_check.py`, per-result content hash); (C) a pre-commit hook BLOCKS stale generated blocks / fingerprints (`--no-verify` escape). Do NOT fingerprint process docs/spec.

**Anchor & tracer docs drift the worst (2026-06-21 follow-up).** Two doc classes silently rot precisely because the close-out gate doesn't cover them: **(1) always-in-context anchors (`CLAUDE.md`)** — it had *inverted framing* (led with the dead autonomous-loop/kickoff/summary_block regime as primary; the actual interactive model was a footnote) **and** a hand-copied program snapshot frozen at 2026-06-18 — the exact copy-drift this system fights, sitting in the most-read file. Fix = **lead with the CURRENT operating mode; replace any hand-copied state with a pointer to the generated `program_state.json`/§0.3.** **(2) tracer docs not in the close-out set (`REPRODUCIBILITY.md`)** — frozen since the baseline commit, its per-experiment table fell behind (missing D1/B1/D-D1-2). Fix = **add it to the DISCIPLINE §1.1 close-out tracker list** so a new experiment must add its row. General rule: any doc that *references* evolving state must either be GENERATED, be in the close-out set, or carry only pointers — never a hand-copy. Audit anchors/tracers periodically; `AUTONOMY.md`'s mission also went stale (pointed at a DONE falsifier).

**Why (process):** the original arc was ~15 turns of doc-integrity infra that BOTH reviewers said was displacing the F1 science ([[evidence-over-scaffolding]] firing); this follow-up was operator-DIRECTED anchor cleanup (legitimate, not self-initiated drift) — but the same caution applies: the runnable F1 science (compaction-at-scale falsifier / 7B transfer) stays queued and must not be indefinitely displaced by doc work. **How to apply:** when an infra/process requirement escalates toward "comprehensive / automatic / all", treat it as an ARCHITECTURE decision — stop reflexively stamping/extending, name the science-vs-infra trade-off legibly, run the cross-family review BEFORE committing the repo to a design, build the right-sized version, then STOP (don't hold the science hostage to perfect doc integrity). The maximal literal interpretation is usually the wrong thing. ([[closeout-gate-before-done]])
