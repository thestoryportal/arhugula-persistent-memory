# Fallback That Nullifies The Spec Is Not F1 Closure

Do not frame a product-preserving fallback as F1 closure if it nullifies the
specification's intended separation of Git and the LLM model as persistent memory
for their respective context classes. Such a route may be an engineering escape
hatch or deployment mitigation, but it is not a readiness answer to the in-weight
memory contract when the operator/spec requires that class to live in model
weights.

If a method path fails only as a local harness/transplant, keep the scientific
condition OPEN/BLOCKING and route to the cheapest source-faithful test or another
method-family attempt. For C10, bounded hybrid does not close F1 under the
operator's stated north-star interpretation; official/source-faithful AnyEdit or
another in-weight method remains the scientific route.
