---
title: C10 accept-bounded means hybrid, not general in-weight DB
date: 2026-06-26
decision: D-C10g-strengthlayers <D-C10g-strengthlayers@d691acab>
---

Accepting the C10 limitation does not kill production viability, but it changes
the product contract. The in-weight layer is viable only as a bounded semantic
serving layer: single-token, prior-coherent, or empirically verified values may
live in weights, while project-coined multi-word semantic values must route to
Git, the index, or another authoritative side-store. Do not describe the current
evidence as supporting arbitrary project facts stored directly in weights.
