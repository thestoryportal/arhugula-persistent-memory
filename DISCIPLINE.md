## 0. THE ULTIMATE GOAL ‚Äî the north star (never drift from this)
**Prove ‚Äî or falsify ‚Äî that the "LLM-as-Database" spec is implementable *before* it is built**, and deliver the **F1 spec-implementation-readiness determination** ("ready / not-ready, with conditions").
The thesis: a transformer's FFN can serve as a **physically queryable + editable graph database** ‚Äî knowledge stored **in-weight** (edited via MEMIT), as `.vindex` overlays on a frozen base, queried via LQL, governed (2PC/ledger/validation), and **deployed on CPU**. Spec: `research_and_specs/llm-as-database-v1_2-integrated-spec.md`. **Everything serves F1. Falsification-first: the goal is the TRUTH about buildability, not green checkmarks.**

> **DRIFT CHECK ‚Äî run before any non-trivial action:** *"Does this advance the F1 readiness determination, or a live falsifier the goal depends on (`EXPERIMENT_RUNBOOK.md` ¬ß0.3)? If not ‚Äî stop and say why."* If you cannot tie the action to F1 or a ¬ß0.3 falsifier, you are drifting.

**North-star fallback rule (2026-06-27).** A fallback that preserves an
engineering product while nullifying the spec's intended separation of Git and
LLM-weight persistent memory is **not** an F1 closure. It may be an engineering
escape hatch, a deployment mitigation, or a scoped product route, but it must be
labelled as such. If the operator states that such a fallback violates the
spec's intent, do not recommend it as the readiness path. Instead, keep the
relevant condition OPEN/BLOCKING and route effort to the cheapest faithful test
or method-family attempt that could still satisfy the in-weight memory contract.

### 1.1 LEARN-AS-YOU-GO ‚Äî record back (the other half of context; this is how the agents *improve*)
Context is a **read‚Üîwrite loop**: pull the relevant memory/CORPUS **before**, record results + learnings **after**. Drift and repeated mistakes come from skipping the write side.
- **‚õî DE-DUP NORM (binding, 2026-06-21 ‚Äî the actual cure for perpetual drift):** do **NOT** copy a result's evolving numbers/caveats/tables into many docs (that *manufactures* drift ‚Äî it's what caused k‚â§2‚Üík‚â§1). **Full detail lives in ONE source** (the `CORPUS/NN` writeup / the amendment / `docs/program_state.json`); every other doc carries a **thin self-contained summary = Decision-ID + one-line verdict + pointer**, NOT a restated copy. Two mechanical halves enforce it (reconciled design, Opus advisor + gpt-5.5 cross-family): **(A) GENERATE what's mechanical** ‚Äî structured status (F1 scorecard, current-position/next-actions, governance status) is rendered from `docs/program_state.json` via `python3 tools/render_state.py --write` into `<!-- BEGIN GENERATED:* -->` blocks (edit the JSON, never between markers); **(B) FINGERPRINT what's narrative** ‚Äî `tools/closeout_check.py` content-hashes each result's source span and requires `<D-ID>@<hash>` wherever it's referenced. A **pre-commit hook** (`tools/git_hooks/`, via `core.hooksPath`) BLOCKS commits with stale generated blocks or stale registered fingerprints (escape: `--no-verify`). Honest scope: this **auto-ENFORCES** currency (no stale commit lands) + **auto-UPDATES** the structured/generated content; it cannot auto-write narrative prose, and it only guards REGISTERED edges (unregistered restatements still drift ‚Äî so prefer pointers over copies).
- **After EVERY experiment** (the `EXPERIMENT_RUNBOOK.md` ¬ß0.4 close-out) ‚Äî the **COMPLETE** canonical-tracker set (this list was under-specified before 2026-06-21): write `CORPUS/NN_*.md` (the ONE detailed source) ‚Üí update `docs/program_state.json` then `python3 tools/render_state.py --write` (refreshes the generated status blocks in PROGRESS ‚ë§ / runbook ¬ß0.3 / checkpoint / bootstrap / README) ‚Üí update `CORPUS/00_MASTER_EVIDENCE.md` + `CORPUS/03_STATUS_LEDGER.md` + `CORPUS/01_PROVENANCE_MANIFEST.md` + `CORPUS/02_VANDV_CHAIN.md` ‚Üí runbook **¬ß12 + ¬ß13** (+ ¬ß5 Decision-ID) ‚Üí `docs/HYPOTHESIS_REGISTER_*.md` ‚Üí `EVIDENCE_INDEX.md` + `docs/EXPERIMENT_REGISTRY.md` ‚Üí **`docs/SPEC_EXPERIMENT_OVERLAY.md` (if the result bears on a spec section ‚Äî add/update its row, pointer not copy)** ‚Üí **`REPRODUCIBILITY.md` re-run table (if the experiment has a runner ‚Äî add its row so the table stays current)** ‚Üí memory + `memory_mirror/` (+ `MEMORY.md` index). **Carry pointers, not copies** (de-dup norm above). Register the result's fingerprint source span in `tools/closeout_fingerprints.json` + stamp `<D-ID>@<hash>` in its narrative references.
  - **üö¶ CLOSE-OUT GATE (binding, mechanical ‚Äî NOT the operator's job to verify):** the experiment is **NOT done** until `python3 tools/closeout_check.py <D-ID>` reports **‚úÖ ALL GREEN**. ## 2. DEEP-THINKING-ON-FAILURE ‚Äî the rigor protocol (mandatory on trigger)
**Triggers (ANY of these ‚Üí run the protocol, do not push past it):**
- an experiment **FAILS / is PARTIAL / returns a surprising or "too-clean" result**;
- you are **stuck / looping / the approach isn't converging**;
- **before declaring any result "conclusive"** or writing it to `CORPUS/`;
- you suspect a **confound or an over-claim**.

**The protocol (rigor + novel, creative-but-GROUNDED, non-hallucinatory ideation):**
1. **Reflect honestly** ‚Äî what does the result *actually* show? Separate `EVIDENCE-SHOWS` from `I-INFER`.
2. **Enumerate ‚â•3 hypotheses** for the failure/result ‚Äî *including* "I am wrong / there is a confound."
3. **Ideate widely but grounded** ‚Äî surface non-obvious approaches; question the problem framing itself; reach past our own data when warranted (¬ß3). **No hallucination:** every idea must be checkable, and every external artifact/claim verified *before* effort.
4. **De-confound** ‚Äî design the *cheapest* test that could **overturn** your own finding; run controls before believing it.
5. **`advisor()`** ‚Äî mandatory **before authoring the new test set** and **before re-declaring done**.
6. **Pre-register** the pass/fail criteria **before** the confirming run.
7. **WRITE the surfaced hypotheses (the deep-thinking yield is context ‚Äî record it).** Every hypothesis from step 2/3 goes durably into the living hypothesis register (`docs/HYPOTHESIS_REGISTER_*.md`) with its **additive framing**: `builds-on` (which PROVEN node ‚Äî `PROGRESS.md` ‚ëÝ), `would-advance` (which F1 gap/chain), `status` (OPEN/TESTING/PROMOTED/KILLED/PARTIAL), and `source` (this ¬ß2 pass / advisor / autoresearch / Perplexity). **A hypothesis not written did not happen** (the ¬ß1.1 read‚Üîwrite rule ‚Äî hypotheses are pulled *back* before choosing the next move; see ¬ß1 "what's proven vs open"). On test, **promote** it into `PROGRESS.md` ‚ëÝ or **kill** it into ‚ë£ with the Decision-ID. This is how a failure's most valuable output (a new direction) survives the session instead of evaporating.

**Non-hallucination rules (always):** cite an artifact or label `UNVERIFIABLE`; mechanics ‚âÝ contract; design-viability ‚âÝ empirical evidence; verify external repos/IDs/claims before building on them; reserve confidence for runs that can fail.

### 3.1 AUTONOMOUS (no-HIL) RUNS ‚Äî extra bounds (`AUTONOMY.md`, `tools/autonomy_driver.py`)
When running unattended, **all of ¬ß0‚Äì¬ß3 still bind**, plus:
- **Hard wall-clock** (`--budget-min`) is a real deadline; **preflight** (`codex_context_guard preflight`) hard-gates before any run. NOT-READY ‚Üí abort.
- **Pre-registered deterministic labels only** ‚Äî PASS/PARTIAL/FAIL/INVALID from a fixed rule on the result JSON (no model judgment). `FAIL` = tested-and-negative = a **real result**; `INVALID` = couldn't evaluate / guard failed = **not** a result. Thresholds are frozen **before** launch; changing them after seeing results voids pre-registration.
- **Per-unit loop bound** (`max_loops`) ‚Äî a retry only fixes a *transient* failure; never loop to chase a different label on a deterministic run.
- **Point at a FALSIFIER, not the optimizer.** A `fenced:true` unit (autoresearch/sweep) may **log candidates only** ‚Äî a fenced PASS is a *lead*, never a conclusion, never written to `CORPUS/`.
- **STAGING, not canonical.** Unattended findings ‚Üí `logs/pending_findings/NN_<unit>.md` with an obligation block. The driver/agent must **NOT** author `CORPUS/*` or edit the ledger/runbook/checkpoint; the operator folds them in on supervised review (¬ß0.4).
- **Standing-auth** (model pulls / cov-compute / disk) PRE-APPROVED + logged; **paid-provider + credentials GATED** ‚Üí `logs/autonomy_gates.jsonl`, skipped, never block the whole run.
- **Self-review ‚âÝ advisor-review.** Unattended "review" by the run model is a self-consistency check; real independence needs a **different model family** (`tools/claude_advisor.sh` for Codex-led work, GPT/Codex for Claude-led work) and must not raise confidence on its own.
